## Evaluator Q&A

### Is this a chatbot?

No. It is a judgment and verification layer that can sit around agents and memory systems.

### Is this RAG?

No. Retrieval can be part of a workflow, but HUQAN is about judgment, admission, and verification rather than just retrieval.

### Is this causal AI?

Not as a statistical or SCM platform. It can represent explicit relations, but the positioning is deterministic judgment, not a generic causal AI claim.

### Does it use an LLM as judge?

It can verify LLM outputs, but the core claim is deterministic judgment and receipt-oriented verification, not a vague LLM-as-oracle setup.

### What is deterministic?

The supported judgment, the verification path, and the readiness evidence recorded in the repo.

### What is local-first?

The system is designed to run with local repo state, local rules, and local verification boundaries before any broader product claims.

### What is ready today?

The verified lines already completed in the repo: relation extraction checkpoint, S1 hardening, README/public positioning alignment, GUV-2, ING-2, release/readiness checkpoint, and the release notes draft.

### What is not implemented yet?

V4 / Workbench, Self-Healer runtime automation, and any claim of complete future risk elimination.

### What is the next roadmap step?

Demo packaging, then any investor-facing or evaluator-facing presentation material that stays within the verified state.
