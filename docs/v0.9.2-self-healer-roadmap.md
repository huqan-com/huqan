# v0.9.2 Self-Healer Loop Roadmap

> Canonical phase sequence and maturity authority for Self-Healer.
>
> Architecture authority: `docs/ADR-007-self-healer-loop.md`.
>
> Runtime contract authority: `lib/self-healer/*` and
> `test/self-healer-*.test.js`.

## Goal

Create a safe, local-first, human-reviewed Self-Healer Loop for HUQAN / AXIOM.

## PR Sequence

### SH-0 - ADR / Blueprint

Maturity: COMPLETE

Docs-only.
Defines architecture, safety model, invariants, PR sequence.

### SH-1 - Finding Schema

Maturity: IMPLEMENTED

Adds schema for:

- finding id
- severity
- confidence
- affected files
- evidence
- suggested tests
- suggested fix
- receipt link

No runtime action yet.

### SH-2 - Audit-Only Report Helper

Maturity: PARTIAL

Adds local audit report helper in `audit_only` mode.

Current boundary:

- accepts caller-provided checks
- normalizes them into validated findings
- does not scan the repository autonomously
- does not run tests
- does not inspect Git history
- does not create branches, patches, memory writes, or PRs

### SH-3 - Finding Classifier

Maturity: IMPLEMENTED

Classifies findings:

- bug
- security
- flaky test
- stale docs
- unsafe pattern
- release hygiene

### SH-4 - Fix Proposal Generator

Maturity: PLANNED

Generates fix plan and test plan.
Does not patch code yet.

### SH-5 - Draft Patch Mode

Maturity: PLANNED

Can create patch in a controlled branch.
No merge.
No broad stage.
No runtime artifacts.

### SH-6 - Draft PR Mode

Maturity: PLANNED

Can open draft PR with:

- finding receipt
- fix plan
- test plan
- risk summary
- rollback note

### SH-7 - Memory/Audit Integration

Maturity: PLANNED

Stores approved Self-Healer events in Memory Core.
No review-less canonical memory write.

### SH-8 - Final Smoke / Docs

Maturity: PLANNED

Documents safe usage and limitations.
Runs full suite.

## Operating Modes

- `audit_only` - IMPLEMENTED
- `proposal_only` - PLANNED
- `draft_patch` - PLANNED
- `draft_pr` - PLANNED
- `blocked` - PLANNED AS EXPLICIT REPORT STATE

## Required Review Gates

- code change requires human review
- security fix requires human review
- memory write requires admission
- draft PR requires approval before merge
- failed tests block progression

## What Success Looks Like

A user can run Self-Healer in dry-run mode and get:

- findings
- severity
- evidence
- suggested fix (planned)
- suggested tests (planned)
- Trust Receipt (planned; current runtime carries nullable `receiptId` only)
- no uncontrolled repo mutation

Current determinism claim is limited to deterministic canonical IDs and
normalization. Full report objects are not guaranteed to be bit-for-bit
identical because timestamp fields may default to current time.

## What It Must Not Do

- no auto-merge
- no silent memory mutation
- no destructive commands
- no secret exfiltration
- no broad refactor
- no unreviewed PR merge
- no production deploy
