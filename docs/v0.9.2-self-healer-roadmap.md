# v0.9.2 Self-Healer Loop Roadmap

## Goal

Create a safe, local-first, human-reviewed Self-Healer Loop for HUQAN / AXIOM.

## PR Sequence

### SH-0 - ADR / Blueprint

Docs-only.
Defines architecture, safety model, invariants, PR sequence.

### SH-1 - Finding Schema

Adds schema for:

- finding id
- severity
- confidence
- affected files
- evidence
- suggested tests
- suggested fix
- receipt link

No runtime action yet.

### SH-2 - Audit Runner Dry Run

Adds local audit runner in `audit_only` mode.
No file writes except explicit report output.
No PR creation.

### SH-3 - Finding Classifier

Classifies findings:

- bug
- security
- flaky test
- stale docs
- unsafe pattern
- release hygiene

### SH-4 - Fix Proposal Generator

Generates fix plan and test plan.
Does not patch code yet.

### SH-5 - Draft Patch Mode

Can create patch in a controlled branch.
No merge.
No broad stage.
No runtime artifacts.

### SH-6 - Draft PR Mode

Can open draft PR with:

- finding receipt
- fix plan
- test plan
- risk summary
- rollback note

### SH-7 - Memory/Audit Integration

Stores approved Self-Healer events in Memory Core.
No review-less canonical memory write.

### SH-8 - Final Smoke / Docs

Documents safe usage and limitations.
Runs full suite.

## Operating Modes

- audit_only
- proposal_only
- draft_patch
- draft_pr
- blocked

## Required Review Gates

- code change requires human review
- security fix requires human review
- memory write requires admission
- draft PR requires approval before merge
- failed tests block progression

## What Success Looks Like

A user can run Self-Healer in dry-run mode and get:

- findings
- severity
- evidence
- suggested fix
- suggested tests
- Trust Receipt
- no uncontrolled repo mutation

## What It Must Not Do

- no auto-merge
- no silent memory mutation
- no destructive commands
- no secret exfiltration
- no broad refactor
- no unreviewed PR merge
- no production deploy
