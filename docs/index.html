HUQAN / AXIOM surface policy

Choose the right surface.

AXIOM ships with one static public demo, one local backend-connected developer UI, and one docs entry page. This page exists only to route visitors to the right surface without creating a competing third product demo.

Public demo

`demo/index.html`

Canonical backend-free concept landing. Safe for static hosting and investor/demo sharing.

  • Static simulation only
  • No API keys or live verification
  • Use for Pages/Vercel/Cloudflare static deploys
Open static demo
Local product UI

`public/index.html`

Canonical backend-connected developer surface served by `node server.js` on localhost.

  • Real graph, trust, and verification flows
  • Requires local server
  • Not intended as static public landing
Local install guide README
Docs entry

`docs/index.html`

Lightweight chooser only. It points to the demo and the install/docs path instead of acting like another product page.

  • Not a third UI mode
  • Not a second static demo
  • Not a backend-connected app
Surface policy
Deploy target Use `demo/index.html` for static public previews and shares.
Developer target Use `public/index.html` via `node server.js` for the local connected UI.
Policy result PTD-2 removes surface ambiguity without starting a UI rewrite.
