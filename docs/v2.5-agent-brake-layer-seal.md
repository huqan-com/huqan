## Safety Invariants

### Global Invariants (All Gates)

1. **Risk classification before execution** — every action is classified by AB1 before any tool call
2. **Tool call authorization before dispatch** — AB2 gates every outbound tool call
3. **Code change verification before commit** — AB3 verifies all code mutations
4. **Memory mutation approval before write** — AB4 approves all memory writes
5. **Automation safety review before deploy** — AB5 reviews all automation workflows
6. **Sandbox isolation before execution** — AB6 evaluates isolation for untrusted code

### Fail-Safe Invariants

7. **Unknown → block** — any unrecognized action, tool, or runner defaults to block/quarantine
8. **Malformed → block** — invalid input shape is rejected, not interpreted
9. **No gate executes runtime operations** — all gates are pure classifiers/advisors
10. **No gate mutates state** — gates classify and advise; they do not write
11. **Deterministic output** — same input always produces same output
12. **Frozen enums** — risk categories, action types, decisions are constant
