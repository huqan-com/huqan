# AXIOM Competitive Positioning

## Position

AXIOM is a local-first reasoning and verification layer for builders, agents, and memory-backed workflows.

It is not a model wrapper, not a chat app, and not a generic orchestration shell. It verifies claims, records receipts, and keeps risky actions behind deterministic gates.

## What AXIOM Is Good At

- Deterministic claim verification
- Contradiction detection
- Provenance-backed trust receipts
- Local-first graph and memory workflows
- Safety gating for risky actions

## What AXIOM Is Not

- Another LLM
- A probabilistic guardrail layer
- A cloud-only SaaS dependency
- A full autonomous agent platform
- A marketing wrapper around model outputs

## Competitive Contrast

- Against LLM-only systems: AXIOM is deterministic and auditable.
- Against guardrail-only systems: AXIOM stores graph state and reasons over it.
- Against generic agent platforms: AXIOM keeps the trust boundary explicit and testable.

## Public Language

Use:

- `local-first reasoning and verification layer`
- `deterministic judgment`
- `claims, receipts, and gates`
- `trusted only when proven`

Avoid:

- `guarantees truth`
- `fully autonomous`
- `enterprise-grade governance` unless specifically proven
- `another AI assistant`
- `chat with your data`
