# AXIOM v0.6 Release Notes

## Summary

v0.6 is the productization and shield release. The goal is not new core features, but turning the existing symbolic engine into a clearer product surface with a safer LLM boundary, cleaner ingest routing, and SDK wrappers for external AI systems.
This release notes file covers the final promotion from `v0.6.0-rc.1` to `v0.6.0`.

## Delivered

### PR-0 Surface Map

- Existing server/product surfaces were mapped before refactoring work began.

### PR-1 Public Index Extraction

- The D3-heavy HTML was extracted out of `server.js` into `public/index.html`.
- The `/` route behavior stayed intact.

### PR-2 Shield Layer

- `/llm-sor` was routed through `lib/shield.js`.
- Shield labels are explicit:
  - `graph-backed`
  - `llm-assisted`
  - `unsupported`
  - `contradicted`
- `autoLearn` defaults to `false`.

### PR-3 Ingest Extraction

- `/api/ingest` routing was moved into `lib/ingest.js`.
- Source handling remains:
  - `github` / `repo`
  - `markdown`
  - `manual`
  - `decision`

### PR-4 Product UI / Graph Polish

- The UI now presents:
  - `Fikrini Yargılat`
  - `Şeytan'ın Avukatı`
  - `Geçmiş Çelişkiler`
  - `Hafıza / Graph`
- Graph metadata is surfaced in the UI.

### PR-4.5 Demo Smoke

- `docs/demo-v0.6.md` documents the smoke flow for the release surface.

### PR-5 SDK Wrappers

- `lib/sdk.js` exposes dependency-free wrappers.
- Supported commands:
  - `verify`
  - `reason`
  - `mri` / `ideaMRI`
  - `devil` / `devilAdvocate`
  - `contradictions` / `contradictionAlert`
  - `shield`
- `LlamaIndex` is intentionally out of scope for v0.6 PR-5.

## Test Count

- `348/348` passing at the end of the PR-5 line.

## Breaking Changes

- No breaking API change was intended for the exposed HTTP surfaces.
- The refactor moves logic out of `server.js` into dedicated modules without changing the route contracts.

## Known Limitations

- `llm-assisted` mode remains conservative by design.
- `companyMode` and `temporal` dependent plugins are still capability-gated.
- `LlamaIndex` integration is intentionally out of scope for this release.
- The product still relies on local-first storage and is not a hosted SaaS runtime.

## Release Position

- Final tag: `v0.6.0`
- Promoted from `v0.6.0-rc.1`
